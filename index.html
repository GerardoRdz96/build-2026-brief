The Agent Stack

isolation · runtime · identity · security
FEATURED

Agents become first-class infrastructure

Not a feature bolted onto an app — the thing that runs the work, with an OS and a control plane built around it.

Microsoft framed an end-to-end agent platform: an OS-native isolation layer, a hosted runtime, a governance plane, and a security harness. The pieces below are the spine of the keynote.

item 09 · [64:07]

MXC — Microsoft Execution Containers

OS-native policy and isolation for autonomous agents on Windows: process-level, session-level, and (for maximum isolation) Windows 365 for agents. The sandbox that makes letting an agent touch your machine safe.

OS-level agent isolation
item 12 · [86:14]

Foundry as an agent platform

A runtime for long-running agents: memory, micro-VM-per-session sandboxes, apply-once guardrails (e.g. PII), auto-generated rubric evaluators, and an agent optimizer that tunes instructions/tools and scores candidates. A LangGraph agent was deployed on stage.

evals + optimizer as platform
item 13 · [85:42]

Agent 365 — the control plane

Per-agent identities and access controls, extended Defender for real-time defense and Purview for compliance. Multi-cloud (AWS, GCP), framework-agnostic. SDK GA, expanding to local Windows agents.

identity + governance for agents
item 11 · [76:26]

GitHub Copilot app

A dev + ops home base: parallel agent sessions via git worktrees, an agent-merge that babysits a PR through CI/review/merge, multi-model under one subscription, and cross-model review (one model checks another's code). Paired with the Rayfin SDK + a Replit partnership.

parallel agents · cross-model review
item 14 · [92:19]

MDASH — security harness

A multi-agent security system orchestrating ~100+ specialized agents across frontier + custom models to find exploitable and AI-specific bugs single models miss. Standalone CLI + inside the Copilot app; defender details / fix; GitHub Advanced Security integration.

100+ agents hunting bugs
item 10 · [65:40]

Open Claw native on Windows

Runs natively on Windows via MXC, with a companion app, sandboxed tool calls, granular folder/clipboard/internet permissions, and a pluggable harness (Copilot, Codex, …). Peter Steinberger announced the nonprofit Open Claw Foundation — "run it inside your company now."

sandboxed · pluggable harness
item 15 · [101:08]

Copilot super app + Autopilots

By summer, chat + Cowork + code in one Copilot app. Autopilots are autonomous, in-tenant, compliant agents with a name, personality, connectors and memory. First out: Scout (Teams chats + Outlook threads).

in-tenant autonomous agents
item 06 · [37:49]

Project Solara — agent-first devices

A turnkey platform for purpose-built devices that extend your agent: a stationary desk device (Windows Hello sign-in) and a wearable "access badge" (fingerprint, camera, mics, hands-free docs). AOSP-based, bring-your-own-agent. Explorers: Best Buy, Target, Accenture.

hardware for agents

Reading the room

Five throughlines

Strip the product names and the same handful of ideas show up again and again across the keynote.

Governance as a product

MXC, Agent 365, Foundry guardrails — agent isolation, identity, and compliance are shipping as first-class platform features, not afterthoughts.

Evals are graduating

Auto-generated rubric evaluators plus an optimizer. Measuring whether an agent is actually good is moving into the core loop.

Frameworks welcome

A LangGraph agent was deployed to Foundry live on stage — the existing open agent frameworks sit squarely inside the platform story.

Cross-model review

One model reviewing another model's code showed up as a real workflow. Different perspectives catch mistakes a single one misses.

Your data is the moat

Frontier Tuning's thesis: the base model isn't the differentiator. What you ground it in and how you keep improving it on your own work is.

MCP everywhere

Web IQ is MCP-native and Foundry exposes a single MCP toolbox — the open tool-calling standard is now load-bearing.
